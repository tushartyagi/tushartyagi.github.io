HTTPS to SSH

Published: Tue 03 March 2015
By Tushar Tyagi

In git.

tags: git basics

One of the biggest pain-points that I've faced after using HTTPS urls for remote repos is that I have to keep providing the credentials after every push. It's okay when you push once or twice in a day, but anything more than that becomes frustrating really fast, especially if the password is like correct horse battery staple.

After changing in repo's directory, using git remote -v prints the urls we're using

$ git remote -v
origin  https://github.com/tushartyagi/tushartyagi.github.io.git (fetch)
origin  https://github.com/tushartyagi/tushartyagi.github.io.git (push)

The ssh url is of the form "git@provider:username/reponame.git" which, for the above repo becomes: git@github.com:tushartyagi/tushartyagi.github.io.git

Using git remote set-url command, we provide the new url for the branch. Since this is a userpage and these are hosted in origin branch, we will change the remote url of origin branch:

$ git remote set-url origin git@github.com:tushartyagi/tushartyagi.github.io.git

The changes will be done, use the remote -v command to verify:

$ git remote -v
origin  git@github.com:tushartyagi/tushartyagi.github.io.git (fetch)
origin  git@github.com:tushartyagi/tushartyagi.github.io.git (push)

Now enjoy the convinience of using ssh -- entering a password only a single time.
